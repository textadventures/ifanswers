IF Answers exists because I've believed for some time that it would be great if we had a Stack Exchange site for Interactive Fiction - i.e. a site where questions and answers can be voted on and edited by the community (like Stack Overflow).

There have been a couple of attempts at getting an Interactive Fiction Stack Exchange site set up, but they didn't get enough traction to get going.

It's kind of a chicken-and-egg thing really. I think an IF Q&A site will only gain momentum after it starts existing - before that happens, there aren't enough of us who really "get" the idea yet to get one started via the Stack Exchange Area 51 process. (And even though I happen to be an employee of Stack Exchange myself, I have no influence over what sites get set up).

Fortunately, there's an open source platform called Question2Answer which allows you to create Stack Exchange-style sites, so that's what I've set up here at ifanswers.com.

Things that are on-topic here:

Technical questions about interactive fiction development tools such as Inform, Twine, Quest, QuestKit, Squiffy, Adrift, TADS etc. are all on-topic here.

Non-technical questions about interactive fiction are also on-topic. These questions could be about general IF design, specific games, entering the IF Comp etc.
